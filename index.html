JavaScript

La evolución de JavaScript consiste en la mejora de sus características, a medida que los navegadores ofrecían un mejor soporte y funcionalidades avanzadas para el control de los elementos de la página. Por ello, debemos tener cuidado en el mercado de los navegadores y la compatibilidad de las funciones que queremos usar.

La evolución de del lenguaje se da en dos lados:

Versiones del lenguaje: En este caso, se muestra el lenguaje ES5 que es el lenguaje que funciona en todos los navegadores.

  • JavaScript 1: Nació con el Netscape 2.0 soportado gran cantidad de intrusiones y funciones.
  • JavaScript 1.1: Se diseño con la llegada de los navegadores 3.0 Implementaba poco mas que su anterior versión, como por ejemplo el tratamiento de imágenes dinámicamente y la creación de arrays.
  • JavaScript 1.2: La versión de navegadores 4.0 tiene la desventaja que es distinta en las plataformas de Microsoft y Netscape, ya que ambos crecieron de distinto modo y en plena lucha en el mercado.
  • JavaScript 1.3: Se implemento en navegadores de versión 5.0. Se limito en algunas diferencias y esperanzas entre dos navegadores.
  • JavaScript 1.5: Versión de navegación actual en el que implementa Netscape 6.
  • Microsoft también evoluciono hasta presentar la versión 5.5 de JScript.
  • Durante 2009 se publica la versión ECMAScript 5. La versión que vamos a tratar principalmente en el manual.
  • En 2015 se da la sexta versión de ECMAScript. Esta versión se puede utilizar en cualquier versión, excepto Internet Explorer, es utilizado por los desarrolladores profesionales, incluso se traduce el código para soporte a navegadores antiguos.

Formas integrar JS a HTML

Una de las formas de integrar el código JS a HTML es por medio del mismo archivo, utilizando la etiqueta “Script” con el que podemos incluir en el inicio del código HTML (entre las etiquetas head), o el cuerpo del código.

Otro método es unir el código de JS por medio de otro archivo. De la forma en el que unimos una hoja de estilos lo podemos aplicar en JS, con la diferencia que utilizamos la etiqueta script con el atributo SRC con el que ubicamos el archivo script. Este método es el más recomendado y útil en caso de reutilizar scripts.

Manejo de Variables, Datos, Expresiones

Variables:

Tales como en la mayoría de leguajes, es necesario manejar información o datos, esto se puede dar por medio de variables en el que se reutilizan, controlan y cambian la información que se den en ellas. Estos se controlan según el lenguaje, en caso de JS se definen con Var, const, let.

Var: Una de las formas más usadas para declarar variables a inicios del lenguaje. Las variables de este tipo pueden ser declaradas sin datos o pueden ser reasignadas, Pero no pueden utilizarse luego de declararse en una función.

Const: Este es igual a las variables definidas como var, con la diferencia de que var tiene el alcance de función y const de bloque, por lo que, si definimos una variable const dentro de una función, no podemos utilizarlo fuera del mismo. Tampoco podemos reasignar los datos de las variables.

Let: Las variables definidas con let, funcionan en bloque comoconst, pero pueden ser reasignadas como la variable var.

Tipo de datos:

String: Es una cadena de caracteres al que se le asigna una posición, desde cero y contando.

Number: No tiene un tipo de numero específico para los números enteros y de coma.

Boolean: Representan dos entidades lógicas, (verdadero o falso).

Null: valor nulo.

Unindefined: Una variable el cual no se le asigno un valor.

Symbol: Consiste en un dato cuyos valores son únicos e inmutables, estos pueden ser utilizados como identificadores de las propiedades de los objetos.

Objet: Es un valor al que podemos usar como identificador para otro valor o memorial.

Expresiones regulares:

Las expresiones regulares son tipos de patrones encerrados entre diagonales con los que utilizamos para encontrar una determinada combinación de caracteres dentro de un texto. Estos se emplean para buscar y remplazar un patrón de una cadena o para la validación de campos en un formulario.

Declaración de funciones

En el lenguaje de JS las funciones son un objeto, por lo que se les puede asignar variables u otros objetos más.

Function Declaratión: Este creara una palabra reservada para la función seguido por un identificador, entre sus paréntesis se declaran los parámetros a usar y entre corcheas el contenido o función que tendrá.

Function Expression: La sintaxis para crear esta función es similar a la función de declaración, con la diferencia de que la definición de la función no inicia por instrucción. Esta función se puede almacenar una variable, permitiéndonos utilizarlo.

Immediately Invoked Function Expression: Este tipo de función nos permite llamarlo en una ocasión inmediatamente, pero no son accesibles posteriormente. Este tipo de función se crea en un ordenador de agrupación, seguido de corchetes y entre llaves el código, permitiendo el interpretado directamente en el monitor de JS.

Shorthand method definition: Esta forma de crear las funciones puede ser utilizada para declaración de un objeto de clases de ES6. Este tipo de función se da por un nombre y sus parámetros indicados entre paréntesis, luego entre llaves el cuerpo de las instrucciones.

Arrow function: Esta función es una de las más resientes de ES6 en la que se compone por una sintaxis diferentes a las anteriores, una de las características de la función es que no crea su propio contexto al ejecutarse, a diferencia de las funciones arrow y expression. Este se definen sus parámetros entre paréntesis, seguidos de un signo igual y mayo a (=>) y los corchetes en el que se indica sus funciones.

Generator function: Como característica principal de esta función a diferencia de las anteriores es que nos permite detenernos o salir dentro de sus instrucciones. Su composición es igual a la función declaration o expression con la diferencia que se utiliza un asterisco al inicio de la función.

Function Constructor: Consiste en una función el cual podemos almacenar una variable, de esta forma podemos llamar la función en caso de necesitarlo.

Operadores:

Operadores de despacho: Ajusta un valor al operario izquierdo en el valor de su operario derecho, El operador de ajuste sigue siendo igual.

Operador de comparación: El operador de comparación se encarga de comparar a sus operados, devolviendo un valor lógico en función de si la comparación es verdadera o falsa. Los operados pueden ser valores numéricos, de cadena, lógicos u objetos.

Operadores aritméticos: El operador aritmético se encarga de tomar valores numéricos o valores de una variable como operarios y devuelve un valor numérico.

Operadores bit a bit: este operador ve a sus operadores como un conjunto de 32 bits en lugar de números decimales. Estos realizan las operaciones representado los valores de forma binaria, y devuelven valores numéricos.

Operadores lógicos: Normalmente, estos operadores se utilizan con valores Booleanos y devuelven el mismo, aun así, los operadores && y || devuelve uno de los valores especificados.

Operadores de cadena: Además de los operadores de comparación el operador de concatenación, conecta dos valores de cadena, devolviendo una cadena que es la unión de ambas.

Operador condicional (ternario): Es el único operador que toma tres operadores, según la condición, el operador toma uno de los valores.

Operador coma: Evalúa ambos operados, devolviendo el ultimo operado. Es necesario utilizarlo por medio de un for.

Operadores unarios: Consiste en una operación realizada con un solo operando.

Operador de agrupación: Permite controlar la precedencia de la evaluación en las expresiones por medio de paréntesis. Definiendo que operación se realiza primero.
